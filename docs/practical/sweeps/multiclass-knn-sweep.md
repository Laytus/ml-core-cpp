# Multiclass Sweep – KNNClassifier

## Setup

This sweep studies the behavior of `KNNClassifier` on the real multiclass dataset:

```txt
data/processed/wine.csv
```

The target is:

```txt
class
```

The sweep results were exported to:

```txt
outputs/practical-exercises/multiclass-classification/hyperparameter_sweep.csv
```

The dataset contains three classes, and the features were standardized before training.

The sweep varied:

```txt
k
```

The main metrics used for interpretation were:

```txt
accuracy
macro_precision
macro_recall
macro_f1
```

Macro-averaged metrics are useful here because they evaluate performance across all classes rather than only reporting global accuracy.

## What changed

The sweep compared different values of the number of neighbors:

```txt
k = 1
k = 3
k = 5
```

The best macro F1 was obtained with:

```txt
run_id: knn_k5
k: 5
macro_f1: 0.973909131804
macro_precision: 0.966666666667
macro_recall: 0.983333333333
accuracy: 0.972222222222
```

The smaller values also performed very well:

```txt
knn_k1
knn_k3
```

Both achieved:

```txt
accuracy: 0.972222222222
macro_recall: 0.983333333333
```

## What improved

Increasing `k` to 5 slightly improved macro F1 and macro precision compared with `k = 1` and `k = 3`.

The best observed configuration was:

```txt
run_id: knn_k5
```

It had the best:

```txt
macro_f1
macro_precision
```

This suggests that using a slightly larger neighborhood helped smooth the decision rule without losing recall.

## What worsened

In this sweep, increasing `k` did not hurt accuracy on the tested split.

However, the differences were small because the Wine dataset is relatively well-separated after standardization.

The main limitation is that this dataset is not very difficult for KNN, so the sweep does not show a strong degradation pattern. On harder or noisier datasets, too small a `k` can overfit local noise, while too large a `k` can oversmooth class boundaries.

## Conceptual takeaway

KNN is a distance-based classifier. The `k` parameter controls how local or smooth the decision rule is:

```txt
small k:
  more local
  more sensitive to individual samples
  can overfit noise

larger k:
  smoother decision boundary
  less sensitive to single samples
  can blur class boundaries if too large
```

In this Wine workflow, the classifier performed well across all tested `k` values. This is consistent with a dataset whose classes are fairly separable after feature standardization.

The main lesson is:

```txt
KNN depends strongly on feature scaling and the choice of k.
For well-separated multiclass data, several nearby k values may perform similarly.
Macro metrics help confirm that performance is balanced across classes.
```
