# Regression Sweep – GradientBoostingRegressor

## Setup

This sweep studies the behavior of `GradientBoostingRegressor` on the real regression dataset:

```txt
data/processed/stock_ohlcv_engineered.csv
```

The target is:

```txt
target_next_return
```

The model was evaluated using the standard regression metrics exported to:

```txt
outputs/practical-exercises/regression/hyperparameter_sweep.csv
```

The sweep varied:

```txt
n_estimators
learning_rate
```

The main metrics used for interpretation were:

```txt
mse
rmse
mae
r2
```

## What changed

The sweep compared different combinations of boosting capacity and learning rate.

In this sanity sweep, the best observed configuration was:

```txt
run_id: gb_n5_lr0.03
n_estimators: 5
learning_rate: 0.03
```

with:

```txt
mse  = 0.000530764443451
rmse = 0.0230383255349
mae  = 0.0170641325285
r2   = -0.0114576760095
```

The next best configurations were also relatively small or conservative:

```txt
gb_n10_lr0.03
gb_n5_lr0.1
```

## What improved

The best configuration used a low learning rate and a small number of estimators.

This suggests that, for this quick stock-return workflow, a conservative boosting setup generalized better than more aggressive settings.

The best run improved the error metrics relative to other tested configurations:

```txt
Lowest MSE:  gb_n5_lr0.03
Lowest RMSE: gb_n5_lr0.03
Lowest MAE:  gb_n5_lr0.03
Best R2:     gb_n5_lr0.03
```

## What worsened

Increasing the number of estimators did not automatically improve the result.

For example:

```txt
gb_n10_lr0.03
```

was slightly worse than:

```txt
gb_n5_lr0.03
```

on all reported regression metrics.

A higher learning rate also did not improve the result in this sweep. The run:

```txt
gb_n5_lr0.1
```

was worse than:

```txt
gb_n5_lr0.03
```

on MSE, RMSE, MAE, and R2.

## Conceptual takeaway

Gradient boosting builds an additive model by fitting weak learners sequentially. Increasing the number of estimators or the learning rate can increase model capacity, but it can also make the model more sensitive to noise.

This dataset predicts short-horizon stock returns, which are noisy and difficult to model from simple engineered OHLCV features. In this context, the conservative configuration performed best.

The negative R2 values are also important. They indicate that even the best tested configuration did not explain the test target variance better than a simple mean baseline. This does not mean the workflow failed; it means the practical experiment correctly exposed the difficulty of the task.

The main lesson is:

```txt
For noisy regression targets, more boosting capacity is not automatically better.
Regularized or conservative boosting settings may generalize better.
Regression metrics should be interpreted alongside the difficulty of the dataset.
```
