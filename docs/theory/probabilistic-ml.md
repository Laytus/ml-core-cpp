# Probabilistic ML Essentials

## 1. Purpose of This Document

This document introduces the probability layer behind Machine Learning.

So far, many models in the project have been presented as deterministic mappings:

```txt
features -> prediction
```

But many ML models can also be understood probabilistically:

```txt
features -> probability distribution over possible outcomes
```

This matters because real ML systems often need to reason about:

```txt
uncertainty
confidence
likelihood
noise
model assumptions
conditional probability
posterior probability
```

Phase 9 focuses on the minimum probability foundation needed before Deep Learning:

```txt
maximum likelihood
MAP estimation
probabilistic outputs
uncertainty concepts
Naive Bayes
Gaussian model intuition
```

The goal is not to implement every probabilistic model.

The goal is to make the probabilistic interpretation of ML clear enough that later concepts, including classification confidence and neural-network likelihood losses, feel natural.

---

## 2. Deterministic vs Probabilistic Thinking

A deterministic model gives a direct prediction.

Example:

```txt
input:
    x

model:
    f(x)

output:
    predicted value or class
```

A probabilistic model gives a distribution or probability.

Example:

```txt
input:
    x

model:
    P(y | x)

output:
    probabilities for possible labels
```

For classification, this means:

```txt
P(y = 0 | x)
P(y = 1 | x)
P(y = 2 | x)
...
```

Instead of only saying:

```txt
class = 1
```

a probabilistic model can say:

```txt
class 0 probability = 0.05
class 1 probability = 0.90
class 2 probability = 0.05
```

This is useful because the model communicates both prediction and uncertainty.

---

## 3. Random Variables and Observations

A random variable represents an uncertain quantity.

Examples:

```txt
Y = class label
X = observed feature vector
R = asset return
D = default event
```

An observation is a realized value.

Example:

```txt
Y = 1
X = [2.3, 0.8, 5.1]
R = -0.02
D = 0
```

In probabilistic ML, we often assume the data was generated by some underlying distribution.

The model tries to estimate or approximate that distribution.

---

## 4. Conditional Probability

Conditional probability is the probability of one event given another event.

Notation:

```txt
P(A | B)
```

means:

```txt
probability of A given that B is known
```

In ML, the most important conditional probability is often:

```txt
P(y | x)
```

This means:

```txt
probability of label y given features x
```

A classifier that outputs probabilities is trying to estimate:

```txt
P(class | features)
```

---

## 5. Joint Probability

Joint probability describes multiple variables together.

Notation:

```txt
P(A, B)
```

means:

```txt
probability of A and B happening together
```

For ML:

```txt
P(x, y)
```

means:

```txt
probability of observing features x and label y together
```

The joint distribution can be decomposed in two equivalent ways:

```txt
P(x, y) = P(y | x) P(x)
```

or:

```txt
P(x, y) = P(x | y) P(y)
```

This second form is very important for Naive Bayes.

---

## 6. Bayes’ Rule

Bayes’ rule connects conditional probabilities:

```txt
P(y | x) = P(x | y) P(y) / P(x)
```

$$
P(y | x) P(x) = P(x | y) P(y) \\
P(x, y) = P(x, y)
$$

Interpretation:

```txt
P(y | x):
    posterior probability
    probability of class y after seeing x

P(x | y):
    likelihood
    probability of seeing x if the class were y

P(y):
    prior probability
    probability of class y before seeing x

P(x):
    evidence / normalization constant
    total probability of seeing x
```

Bayes’ rule is the foundation of Bayesian reasoning and Naive Bayes.

---

## 7. Prior, Likelihood, Posterior, Evidence

Bayes’ rule can be read as:

```txt
posterior ∝ likelihood × prior
```

More explicitly:

```txt
P(y | x) = P(x | y) P(y) / P(x)
```

The evidence `P(x)` normalizes the probabilities so they sum to 1 across all possible classes.

For classification, if we compare classes, `P(x)` is the same for all classes.

So we often choose the class that maximizes:

```txt
P(x | y) P(y)
```

This is the core idea behind Naive Bayes classification.

---

## 8. Maximum Likelihood Estimation

Maximum Likelihood Estimation, or MLE, is a method for estimating model parameters.

The idea is:

```txt
choose the parameters that make the observed data most likely
```

Suppose a model has parameters:

```txt
theta
```

and data:

```txt
D = {x_1, x_2, ..., x_n}
```

The likelihood is:

```txt
P(D | theta)
```

This means:

```txt
probability of observing the data if the parameters were theta
```

MLE chooses:

```txt
theta_hat = argmax_theta P(D | theta)
```

In words:

```txt
choose theta that maximizes the likelihood of the observed data
```

---

## 9. Likelihood Is a Function of Parameters

A key point:

```txt
probability:
    treats parameters as fixed and data as variable

likelihood:
    treats data as fixed and parameters as variable
```

Example:

```txt
probability:
    Given coin bias p = 0.5, what is the probability of observing HHT?

likelihood:
    Given observed data HHT, which coin bias p makes this observation most likely?
```

For a coin with probability of heads `p`, observing:

```txt
H, H, T
```

has likelihood:

```txt
L(p) = p * p * (1 - p)
```

MLE tries to find the `p` that maximizes this function.

For this example:

```txt
p_hat = 2 / 3
```

because 2 of the 3 observations are heads.

---

## 10. Log-Likelihood

Likelihoods often multiply many probabilities.

For independent observations:

```txt
P(D | theta) = product_i P(x_i | theta)
```

$$
P(D | \theta) = \prod_i{P(x_i | \theta)}
$$

Products of many small probabilities can become numerically tiny.

So we usually work with log-likelihood:

```txt
log P(D | theta) = sum_i log P(x_i | theta)
```

$$
\log{P(D | \theta)} = \sum_i{\log{P(x_i | \theta)}}
$$

This is easier numerically and mathematically.

Because log is monotonic:

```txt
maximizing likelihood
```

is equivalent to:

```txt
maximizing log-likelihood
```

In ML optimization, we often minimize negative log-likelihood:

```txt
NLL = -log P(D | theta)
```
$$
\text{NLL} = -\log{P(D | \theta)} = -\sum_i{\log{P(x_i | \theta)}}
$$

This connects directly to cross-entropy losses.

---

## 11. MLE and Supervised Learning

Many supervised ML losses can be interpreted as maximum likelihood.

Examples:

```txt
Linear regression with Gaussian noise:
    minimizing MSE corresponds to maximizing Gaussian likelihood

Logistic regression:
    minimizing binary cross-entropy corresponds to maximizing Bernoulli likelihood

Softmax regression:
    minimizing categorical cross-entropy corresponds to maximizing categorical likelihood
```

This is a very important connection.

Loss functions are not arbitrary.

They often correspond to assumptions about how data was generated.

---

## 12. Example: Linear Regression as MLE

Linear regression can be interpreted as Maximum Likelihood Estimation if we assume that the target is generated by a linear function plus Gaussian noise.

The deterministic linear model is:

```txt
prediction_i = w^T x_i + b
```

or:

$$
\hat{y}_i = w^T x_i + b
$$

But real data is usually not perfectly linear. So we assume the observed target contains noise:

```txt
y_i = w^T x_i + b + noise_i
```

$$
y_i = w^T x_i + b + \varepsilon_i
$$

where:

```txt
ε_i = noise for sample i
```

Now we make the key probabilistic assumption:

```txt
noise_i ~ Normal(0, sigma^2)
```

$$
\varepsilon_i \sim \mathcal{N}(0, \sigma^2)
$$

This means the noise is centered around zero and has variance `sigma^2`.

So most errors are expected to be small, and large errors are possible but less likely.

---

### 12.1 Why `y_i | x_i` is Gaussian

The model says:

$$
y_i = w^T x_i + b + \varepsilon_i
$$

For a fixed input `x_i`, the value:

$$
w^T x_i + b
$$

is just a deterministic number.

The only random part is:

$$
\varepsilon_i
$$

Since we assumed:

$$
\varepsilon_i \sim \mathcal{N}(0, \sigma^2)
$$

then adding the deterministic value `w^T x_i + b` shifts the Gaussian mean:

$$
w^T x_i + b + \varepsilon_i \sim \mathcal{N}(w^T x_i + b, \sigma^2)
$$

Therefore:

```txt
y_i | x_i ~ Normal(w^T x_i + b, sigma^2)
```

or:

$$
y_i \mid x_i \sim \mathcal{N}(w^T x_i + b, \sigma^2)
$$

Important interpretation:

```txt
y_i | x_i is not the likelihood by itself.
```

It is the conditional distribution of the target given the input and the model parameters.

More explicitly, we should write:

$$
y_i \mid x_i, w, b, \sigma^2 \sim \mathcal{N}(w^T x_i + b, \sigma^2)
$$

The **likelihood** is the *probability density of the observed value `y_i` under that distribution*, viewed as a function of the parameters `w`, `b`, and sometimes `sigma^2`.

---

### 12.2 Probability density for one observation

The Gaussian density is:

$$
p(y_i \mid x_i, w, b, \sigma^2) =
\frac{1}{\sqrt{2\pi\sigma^2}}
\exp\left(
-\frac{(y_i - (w^T x_i + b))^2}{2\sigma^2}
\right)
$$

Define the prediction error / residual:

$$
r_i = y_i - (w^T x_i + b)
$$

Then:

$$
p(y_i \mid x_i, w, b, \sigma^2) =
\frac{1}{\sqrt{2\pi\sigma^2}}
\exp\left(
-\frac{r_i^2}{2\sigma^2}
\right)
$$

This formula shows why squared error appears.

The **Gaussian density** gives **lower probability** to **large residuals** according to:

$$
\exp\left(-\frac{r_i^2}{2\sigma^2}\right)
$$

So **maximizing the probability of the observed data** means **making squared residuals small**.

---

### 12.3 Likelihood for the full dataset

Assume the observations are independent given the parameters.

Then the **likelihood of the full dataset** is the **product of individual densities**:

$$
L(w, b, \sigma^2) =
\prod_{i=1}^{n}
p(y_i \mid x_i, w, b, \sigma^2)
$$

Substituting the **Gaussian density**:

$$
L(w, b, \sigma^2) =
\prod_{i=1}^{n}
\frac{1}{\sqrt{2\pi\sigma^2}}
\exp\left(
-\frac{(y_i - (w^T x_i + b))^2}{2\sigma^2}
\right)
$$

This is the **likelihood**.

**MLE** chooses the **parameters** that **maximize this expression**:

$$
\hat{w}, \hat{b} =
\arg\max_{w,b} L(w,b,\sigma^2)
$$

Usually, we **maximize** the **log-likelihood** instead because it turns *products into sums*.

---

### 12.4 Log-likelihood

Taking log:

$$
\log L(w, b, \sigma^2) =
\sum_{i=1}^{n}
\log
\left[
\frac{1}{\sqrt{2\pi\sigma^2}}
\exp\left(
-\frac{(y_i - (w^T x_i + b))^2}{2\sigma^2}
\right)
\right]
$$

Using log rules:

$$
\log L(w, b, \sigma^2) =
\sum_{i=1}^{n}
\left[
-\frac{1}{2}\log(2\pi\sigma^2)
-\frac{(y_i - (w^T x_i + b))^2}{2\sigma^2}
\right]
$$

So:

$$
\log L(w, b, \sigma^2) =
-\frac{n}{2}\log(2\pi\sigma^2)
-\frac{1}{2\sigma^2}
\sum_{i=1}^{n}(y_i - (w^T x_i + b))^2
$$

The first term:

$$
-\frac{n}{2}\log(2\pi\sigma^2)
$$

does not depend on `w` or `b` if `sigma^2` is fixed.

The factor:

$$
-\frac{1}{2\sigma^2}
$$

is also constant with respect to `w` and `b`.

Therefore, maximizing the log-likelihood with respect to `w` and `b` is equivalent to minimizing:

$$
\sum_{i=1}^{n}(y_i - (w^T x_i + b))^2
$$

This is the **Sum of Squared Errors**, or **SSE**.

---

### 12.5 Relation to MSE

**MSE** is:

$$
\text{MSE} =
\frac{1}{n}
\sum_{i=1}^{n}(y_i - (w^T x_i + b))^2
$$

**SSE** is:

$$
\text{SSE} =
\sum_{i=1}^{n}(y_i - (w^T x_i + b))^2
$$

Since:

$$
\text{MSE} = \frac{1}{n}\text{SSE}
$$

and `1/n` is a positive constant, **minimizing SSE** is **equivalent** to **minimizing MSE**.

So:

```txt
maximizing Gaussian likelihood
= maximizing Gaussian log-likelihood
= minimizing SSE
= minimizing MSE
```

More precisely, this equivalence holds for estimating `w` and `b` under the *assumption* of **independent Gaussian noise with constant variance**.

---

### 12.6 Final intuition

Linear regression with MSE is not just an arbitrary optimization problem.

It corresponds to this probabilistic story:

```txt
1. The true target is approximately linear in the features.
2. The observed target is corrupted by Gaussian noise.
3. Small errors are more likely than large errors.
4. The most likely parameters are the ones that minimize squared error.
```

That is why:

```txt
MSE minimization = Maximum Likelihood Estimation under Gaussian noise
```

This connection is fundamental because it shows that many ML losses are tied to assumptions about data-generating processes.

## 13. Example: Logistic Regression as MLE

**Logistic Regression** can be interpreted as **Maximum Likelihood Estimation** if we assume that the **target** is generated from a **Bernoulli distribution** whose **probability depends on a linear score**.

This is the classification analogue of the previous linear regression example.

For linear regression, we assumed:

```txt
y_i = linear prediction + Gaussian noise
```

For logistic regression, we assume:

```txt
y_i is a binary random variable with probability controlled by a linear score
```

The target is binary:

```txt
y_i ∈ {0, 1}
```

The model first computes a linear score:

```txt
z_i = w^T x_i + b
```

or:

$$
z_i = w^T x_i + b
$$

Then it maps that score into a probability using the sigmoid function:

$$
p_i = \sigma(z_i) = \frac{1}{1 + e^{-z_i}}
$$

So:

```txt
p_i = P(y_i = 1 | x_i, w, b)
```

or:

$$
p_i = P(y_i = 1 \mid x_i, w, b)
$$

This probability is the model's estimate that sample `i` belongs to class `1`.

---

### 13.1 Why the sigmoid appears

The linear score:

$$
z_i = w^T x_i + b
$$

can be any real number:

```txt
negative infinity to positive infinity
```

But a probability must be between 0 and 1.

The sigmoid function maps real numbers into probabilities:

$$
\sigma(z) = \frac{1}{1 + e^{-z}}
$$

Its behavior is:

```txt
large positive z:
    sigmoid(z) close to 1

z = 0:
    sigmoid(z) = 0.5

large negative z:
    sigmoid(z) close to 0
```

So logistic regression interprets the linear score as evidence for class `1`.

A positive score means class `1` is more likely.

A negative score means class `0` is more likely.

---

### 13.2 Bernoulli model for one target

A Bernoulli random variable models a binary outcome.

If:

```txt
y_i ∈ {0, 1}
```

and:

```txt
p_i = P(y_i = 1 | x_i, w, b)
```

then:

$$
y_i \mid x_i, w, b \sim \text{Bernoulli}(p_i)
$$

This means:

$$
P(y_i = 1 \mid x_i, w, b) = p_i
$$

and:

$$
P(y_i = 0 \mid x_i, w, b) = 1 - p_i
$$

Since logistic regression sets:

$$
p_i = \sigma(w^T x_i + b)
$$

we can write:

$$
y_i \mid x_i, w, b \sim \text{Bernoulli}(\sigma(w^T x_i + b))
$$

Important interpretation:

```txt
y_i | x_i is not the likelihood by itself.
```

It is the conditional distribution of the target given the input and model parameters.

The likelihood is the probability of the observed target value under this Bernoulli distribution, viewed as a function of `w` and `b`.

---

### 13.3 Probability of one observed label

For a single binary label, there are two cases:

If:

```txt
y_i = 1
```

then:

$$
P(y_i \mid x_i, w, b) = p_i
$$

If:

```txt
y_i = 0
```

then:

$$
P(y_i \mid x_i, w, b) = 1 - p_i
$$

These two cases can be written compactly as:

$$
P(y_i \mid x_i, w, b) =
p_i^{y_i}(1 - p_i)^{1 - y_i}
$$

This works because `y_i` is either 0 or 1.

If `y_i = 1`:

$$
p_i^{y_i}(1 - p_i)^{1 - y_i} =
p_i^1(1 - p_i)^0 = p_i
$$

If `y_i = 0`:

$$
p_i^{y_i}(1 - p_i)^{1 - y_i} =
p_i^0(1 - p_i)^1 = 1 - p_i
$$

This compact expression is the **Bernoulli likelihood for one observation**.

---

### 13.4 Likelihood for the full dataset

Assume samples are independent given the parameters.

Then the **likelihood of the full dataset** is the **product of individual Bernoulli probabilities**:

$$
L(w, b) = \prod_{i=1}^{n}
P(y_i \mid x_i, w, b)
$$

Substituting the Bernoulli expression:

$$
L(w, b) = \prod_{i=1}^{n}
p_i^{y_i}(1 - p_i)^{1 - y_i}
$$

where:

$$
p_i = \sigma(w^T x_i + b)
$$

MLE chooses:

$$
\hat{w}, \hat{b} = \arg\max_{w,b} L(w,b)
$$

As with linear regression, it is easier to work with the log-likelihood.

---

### 13.5 Log-likelihood

Taking log of the likelihood:

$$
\log L(w, b) = \log \prod_{i=1}^{n} 
p_i^{y_i}(1 - p_i)^{1 - y_i}
$$

Using log rules:

$$
\log L(w, b) = \sum_{i=1}^{n}
\log\left[p_i^{y_i}(1 - p_i)^{1 - y_i}\right]
$$

So:

$$
\log L(w, b) = \sum_{i=1}^{n}
\left[
y_i \log(p_i) + (1 - y_i)\log(1 - p_i)
\right]
$$

This is the **Bernoulli log-likelihood**.

**Maximum Likelihood Estimation** wants to **maximize this quantity**.

---

### 13.6 Negative log-likelihood and binary cross-entropy

In **ML**, **optimization code** usually **minimizes a loss** instead of maximizing a likelihood.

So we **minimize** the **negative log-likelihood**:

$$
-\log L(w,b) = -\sum_{i=1}^{n}
\left[
y_i \log(p_i) + (1 - y_i)\log(1 - p_i)
\right]
$$

The **average negative log-likelihood** is:

$$
\text{BCE} = -\frac{1}{n}\sum_{i=1}^{n}
\left[
y_i \log(p_i) + (1 - y_i)\log(1 - p_i)
\right]
$$

This is **Binary Cross-Entropy**.

So:

```txt
minimizing binary cross-entropy
= minimizing negative Bernoulli log-likelihood
= maximizing Bernoulli likelihood
```

More precisely, this equivalence holds under the *assumption* that **each binary target** is **generated from a Bernoulli distribution** whose **probability** is given by the **logistic regression model**.

---

### 13.7 Why the BCE terms make sense

For one sample, **BCE** is:

$$
-\left[y_i \log(p_i) + (1 - y_i)\log(1 - p_i)\right]
$$

If the true label is `1`, then:

$$
y_i = 1
$$

and BCE becomes:

$$
-\log(p_i)
$$

So the model is rewarded for making `p_i` close to 1.

If the true label is `0`, then:

$$
y_i = 0
$$

and BCE becomes:

$$
-\log(1 - p_i)
$$

So the model is rewarded for making `p_i` close to 0.

Bad confident predictions are heavily penalized.

Example:

```txt
true label = 1
model predicts p = 0.99:
    loss = -log(0.99), small

true label = 1
model predicts p = 0.01:
    loss = -log(0.01), large
```

This is why BCE is appropriate for probabilistic binary classification.

---

### 13.8 Relation to odds and log-odds

Logistic regression has another important probabilistic interpretation.

The odds of class `1` are:

$$
\frac{p_i}{1 - p_i}
$$

The log-odds are:

$$
\log\left(\frac{p_i}{1 - p_i}\right)
$$

*Logistic regression assumes* that the **log-odds are linear in the features**:

$$
\log\left(\frac{p_i}{1 - p_i}\right) = w^T x_i + b
$$

So each feature contributes linearly to the log-odds of class `1`.

This is one reason logistic regression is interpretable.

A **positive weight increases** the **log-odds of class `1`** when that **feature increases**.

A **negative weight decreases** the **log-odds of class `1`** when that **feature increases**.

---

### 13.9 Final intuition

Logistic regression with binary cross-entropy corresponds to this probabilistic story:

```txt
1. Each sample has a binary target y_i ∈ {0, 1}.
2. The model computes a linear score z_i = w^T x_i + b.
3. The sigmoid maps that score into a probability p_i.
4. The target is assumed to be drawn from Bernoulli(p_i).
5. The most likely parameters are the ones that assign high probability to the observed labels.
```

That is why:

```txt
Binary Cross-Entropy minimization = Maximum Likelihood Estimation under a Bernoulli model
```

This mirrors the linear regression result:

```txt
Linear Regression + Gaussian noise -> MSE
Logistic Regression + Bernoulli targets -> Binary Cross-Entropy
```

## 14. Maximum A Posteriori Estimation

**Maximum A Posteriori estimation**, or **MAP**, **extends MLE** by **adding a prior over parameters**.

MLE chooses:

```txt
theta_hat = argmax_theta P(D | theta)
```

MAP chooses:

```txt
theta_hat = argmax_theta P(theta | D)
```

Using Bayes’ rule:

```txt
P(theta | D) ∝ P(D | theta) P(theta)
```

So MAP chooses:

```txt
theta_hat = argmax_theta P(D | theta) P(theta)
```

In words:

```txt
choose parameters that make the data likely
while also being plausible under the prior
```

---

## 15. MLE vs MAP

**MLE** uses only the **data likelihood**:

```txt
maximize likelihood
```

**MAP** uses **likelihood plus prior**:

```txt
maximize likelihood × prior
```

Comparison:

```txt
MLE:
    data-driven
    no explicit prior preference
    can overfit when data is limited

MAP:
    data-driven plus prior belief
    can regularize estimates
    can be more stable with limited data
```

**MAP** is *especially useful* when **data is scarce or noisy**.

---

## 16. MAP and Regularization

Regularization can often be interpreted as a prior.

Example:

```txt
L2 regularization
```

corresponds to a **Gaussian prior over weights**.

If we assume:

```txt
w ~ Normal(0, sigma^2 I)
```

then large weights are less likely.

The MAP objective becomes equivalent to:

```txt
negative log-likelihood + L2 penalty
```

So **Ridge regression** and **L2-regularized logistic regression** can be *interpreted probabilistically*.

This is a very important bridge:

```txt
regularization = prior belief about parameters
```

For example:

```txt
L2:
    weights should be small and smooth

L1:
    weights should be sparse
```

This connection matters in finance and ML because priors help prevent overfitting unstable patterns.

## 17. Probabilistic Outputs

A probabilistic output estimates a probability distribution instead of only a hard prediction.

Examples:

```txt
Binary classification:
    P(y = 1 | x) = 0.82

Multiclass classification:
    P(y = 0 | x) = 0.10
    P(y = 1 | x) = 0.75
    P(y = 2 | x) = 0.15

Regression:
    y ~ Normal(mean, variance)
```

A hard class prediction can be derived from probabilities:

```txt
predict class with highest probability
```

But the probabilities contain more information.

---

## 18. Confidence vs Uncertainty

A model may output:

```txt
P(class 1 | x) = 0.99
```

This looks confident.

But **confidence is not always the same as correctness**.

A model can be:

```txt
confident and correct
confident and wrong
uncertain and correct
uncertain and wrong
```

Probabilistic outputs are useful only if they are meaningful and reasonably calibrated.

---

## 19. Calibration

**Calibration** describes **whether predicted probabilities match observed frequencies**.

Example:

```txt
Among all samples where the model predicts 0.80 probability,
roughly 80% should actually be positive.
```

If this is true, the model is well calibrated.

If a model often predicts 0.99 but is correct only 70% of the time, it is overconfident.

Calibration is critical in finance, risk, medicine, and decision systems.

In finance, a probability is rarely useful by itself.

It matters because it affects expected value, risk sizing, and decisions under uncertainty.

## 20. Aleatoric and Epistemic Uncertainty

There are two common uncertainty types:

```txt
aleatoric uncertainty:
    uncertainty from inherent randomness / noise

epistemic uncertainty:
    uncertainty from lack of knowledge or limited data
```

Examples:

```txt
Aleatoric:
    market movement caused by genuinely unpredictable events
    noisy measurements
    overlapping classes

Epistemic:
    model has not seen enough similar data
    training set is small
    feature space region is unfamiliar
```

*More data* **can** *reduce epistemic uncertainty*.

*More data* **cannot fully** *remove aleatoric uncertainty*.

---

## 21. Predictive Uncertainty in Classification

For classification, uncertainty often appears as probability spread.

**Example confident prediction**:

```txt
P(class 0 | x) = 0.01
P(class 1 | x) = 0.98
P(class 2 | x) = 0.01
```

**Example uncertain prediction**:

```txt
P(class 0 | x) = 0.34
P(class 1 | x) = 0.33
P(class 2 | x) = 0.33
```

The second output suggests the model sees no clear dominant class.

However, probability spread is not the whole story.

The model may be uncertain because:

```txt
the sample is ambiguous
the sample is out of distribution
the model is poorly trained
the features are insufficient
```

---

## 22. Decision-Making with Probabilities

Probabilities are useful because they support decision-making under uncertainty.

Example:

```txt
P(default | borrower features) = 0.03
```

This does not directly tell us what to do.

A decision requires:

```txt
expected gain
expected loss
risk tolerance
capital constraints
thresholds
```

In finance, a probabilistic prediction is often useful only after converting it into expected value or risk-adjusted decisions.

Example:

```txt
expected value = probability_of_gain * gain - probability_of_loss * loss
```

This is why probabilistic ML is not only about predicting probabilities.

It is about making better decisions under uncertainty.

# Naive Bayes

## 23. Why Naive Bayes Matters

**Naive Bayes** is *one of the simplest probabilistic classifiers*.

It is important because it forces you to understand:

```txt
Bayes’ rule
class priors
likelihoods
posterior probabilities
conditional independence assumptions
generative vs discriminative modeling
probability-based classification
```

Even if Naive Bayes is not always the strongest modern model, it is conceptually fundamental.

It is especially valuable because it shows how classification can be built from probability assumptions.

---

## 24. Naive Bayes as a Generative Classifier

Many classifiers model:

```txt
P(y | x)
```

directly.

These are called **discriminative classifiers**.

Examples:

```txt
Logistic Regression
Linear SVM
Softmax Regression
```

Naive Bayes instead models:

```txt
P(x | y)
```

and:

```txt
P(y)
```

Then it uses Bayes’ rule to compute:

```txt
P(y | x)
```

This makes **Naive Bayes** a **generative classifier**.

It models how the data could be generated for each class.

---

## 25. The Core Naive Bayes Formula

Bayes’ rule says:

```txt
P(y | x) = P(x | y) P(y) / P(x)
```

$$
P(y | x) P(x) = P(x | y) P(y)
$$

For **classification**, we **choose the class with the largest posterior probability**:

```txt
prediction = argmax_y P(y | x)
```

Since `P(x)` is the same for every class, we can ignore it when comparing classes:

```txt
prediction = argmax_y P(x | y) P(y)
```

This is the **basic Naive Bayes classifier**.

---

## 26. Feature Vector Notation

A **sample** has multiple features:

```txt
x = [x_1, x_2, ..., x_d]
```

So the **likelihood** is:

```txt
P(x | y) = P(x_1, x_2, ..., x_d | y)
```

This joint likelihood can be difficult to estimate.

For many features, modeling the full joint distribution is expensive and data-hungry.

Naive Bayes solves this by making a strong simplifying assumption.

---

## 27. The Naive Conditional Independence Assumption

Naive Bayes assumes **features are conditionally independent given the class**.

That means:

```txt
P(x_1, x_2, ..., x_d | y)
=
P(x_1 | y) P(x_2 | y) ... P(x_d | y)
```

So:

```txt
P(x | y) = product_j P(x_j | y)
```

$$
P(x | y) = \prod_j{P(x_j | y)}
$$

This is the **“naive” assumption**.

It is called naive because *real-world features are often not independent*.

But the assumption makes the *model simple, fast, and surprisingly effective in many settings*.

---

## 28. Full Naive Bayes Classification Rule

Using the **independence assumption**:

```txt
prediction = argmax_y P(y) product_j P(x_j | y)
```

$$
\text{prediction} = \argmax_y{P(y) \prod_j{P(x_j | y)}}
$$

In words:

```txt
for each class:
    compute prior probability of the class
    multiply by the likelihood of each feature under that class
choose the class with the highest result
```

Because products of many probabilities can underflow, we usually use logs:

```txt
score(y) = log P(y) + sum_j log P(x_j | y)
```

$$
\text{score}(y) = \log{P(y)} + \sum_j{\log{P(x_j | y)}}
$$

Then:

```txt
prediction = argmax_y score(y)
```

$$
\text{prediction} = \argmax_y{\text{score}(y)}
$$

This **log-score form** is the form you should implement.

---

## 29. Class Priors

The class prior is:

```txt
P(y)
```

It *represents* **how common each class is before observing features**.

A *simple estimate* is **frequency**:

```txt
P(y = c) = count(class c) / n
```

$$
P(y = c) = \frac{\text{count}(\text{class c})}{n}
$$

Example:

```txt
100 samples
70 class 0
30 class 1
```

Then:

```txt
P(y = 0) = 0.70
P(y = 1) = 0.30
```

**In finance, priors matter a lot.**

If defaults are rare, fraud is rare, or extreme market moves are rare, the prior probability can dominate unless evidence is strong.

---

## 30. Likelihoods

The likelihood is:

```txt
P(x_j | y)
```

$$
P(x_j | y)
$$

It asks:

```txt
if the class were y, how likely is this feature value?
```

Different Naive Bayes variants use different likelihood models.

Common variants:

```txt
Gaussian Naive Bayes:
    continuous features modeled with Gaussian distributions

Multinomial Naive Bayes:
    count features, often text word counts

Bernoulli Naive Bayes:
    binary features, often word present/absent indicators
```

For this C++ ML Core project, the most natural implementation is:

```txt
Gaussian Naive Bayes
```

because the project already uses numerical matrix features.

---

## 31. Gaussian Naive Bayes

**Gaussian Naive Bayes** *assumes* that **each feature follows a Gaussian distribution within each class**.

For each class `c` and feature `j`, estimate:

```txt
mean_{c,j}
variance_{c,j}
```
$$
\mu_{c,j} \\
\sigma^{2}_{c,j}
$$

Then:

```txt
x_j | y = c ~ Normal(mean_{c,j}, variance_{c,j})
```

$$
x_j | y = c \sim \mathcal{N}(\mu_{c,j}, \sigma^{2}_{c,j})
$$

So the **likelihood** is:

```txt
P(x_j | y = c)
=
NormalPDF(x_j; mean_{c,j}, variance_{c,j})
```

$$
P(x_j | y = c) = \text{NormalPDF}(x_j; \mu_{c,j}, \sigma^{2}_{c,j})
$$

The model stores:

```txt
class priors
per-class feature means
per-class feature variances
```

---

## 32. Gaussian PDF

The **Gaussian probability density function** is:

```txt
N(x; μ, σ²)
=
1 / sqrt(2πσ²) * exp(-(x - μ)^2 / (2σ²))
```

$$
\mathcal{N}(x; \mu, \sigma^2) = \frac{1}{\sqrt{(2 \pi \sigma^2)}} \exp{\frac{-(x - \mu)^2}{2 \sigma^2}}
$$

where:

```txt
μ = mean
σ² = variance
```

In Gaussian Naive Bayes, for each class and feature:

```txt
μ = mean of that feature among samples in the class
σ² = variance of that feature among samples in the class
```

The model then computes how likely a new feature value is under each class-specific Gaussian.

---

## 33. Gaussian Naive Bayes Score

For a class `c`, the **posterior score** is:

```txt
score(c) = log P(y = c) + sum_j log N(x_j; μ_{c,j}, σ²_{c,j})
```

$$
P(y | x) = P(y) P(x | y)
$$

$$
L(c) = P(y = c) \prod_j{\mathcal{N}(x_j; \mu_{c,j}, \sigma^{2}_{c,j})}
$$

$$
\begin{align*}
    \text{score}(c) &= \log{P(y = c)} + \sum_j{\log{\mathcal{N}(x_j; \mu_{c,j}, \sigma^{2}_{c,j})}} \\
    & = \log{\frac{\text{count}(\text{class c})}{\text{n\_samples}}} + 
    \sum_{j}
    \left[
    -\frac{1}{2}\log(2\pi\sigma^2)
    -\frac{(x - \mu)^2}{2\sigma^2}
    \right]
\end{align*}
$$

Then:

```txt
prediction = argmax_c score(c)
```

$$
\text{prediction} = \argmax_c{\text{score}(c)}
$$

This is the log form of:

```txt
P(y = c) product_j P(x_j | y = c)
```

$$
P(y = c) \prod_j{P(x_j | y = c)}
$$

The log form is preferred because:

```txt
products become sums
small probabilities avoid underflow
numerical stability improves
```

---

## 34. Why Variance Smoothing Is Needed

If a **feature** has **zero variance inside a class**, then:

```txt
σ² = 0
```

$$
\sigma^2 = 0
$$

This **breaks** the **Gaussian PDF** because *it divides by variance*.

Example:

```txt
class 0 feature values:
    [5, 5, 5, 5]

variance:
    0
```

To avoid division by zero, Gaussian Naive Bayes uses **variance smoothing**:

```txt
variance = variance + epsilon
```

where `epsilon` is a **small positive value**.

This makes the *model numerically stable*.

Recommended first implementation:

```txt
epsilon = 1e-9
```

or as an option:

```txt
variance_smoothing = 1e-9
```

---

## 35. Naive Bayes Prediction Probabilities

Naive Bayes can produce **normalized class probabilities**.

First compute log scores:

```txt
score(c)
```

Then apply **softmax over class scores**:

```txt
P(y = c | x) = exp(score(c)) / sum_k exp(score(k))
```

$$
P(y = c | x) = \frac{\exp{\text{score}(c)}}{\sum_k{\exp{\text{score}(k)}}}
$$

For numerical stability:

```txt
subtract max score before exponentiating
```

For one sample, the maximum score is:

```txt
max_score = max over classes c of score(c)
```

$$
\text{max\_score} = \max_{c}\; \text{score}(c)
$$

So:

```txt
stable_score(c) = score(c) - max_score
```

$$
\text{stable\_score}(c) = \text{score}(c) - \text{max\_score}
$$

Then:

```txt
probability(c) = exp(stable_score(c)) / sum_k exp(stable_score(k))
```

$$
p(c) = \frac{\exp{\text{stable\_score}(c)}}{\sum_k{\exp{\text{stable\_score}(k)}}}
$$

This gives:

```txt
predict_proba(X)
```

and:

```txt
predict(X) = argmax probability
```

$$
\text{predict}(X) = \argmax{p}
$$

## 36. Why Naive Bayes Can Work Despite Wrong Assumptions

The **independence assumption is often false**.

**Features** in **real data** are **usually correlated**.

For example:

```txt
income and savings
interest rates and bond prices
volatility and option prices
word frequencies in text
```

Naive Bayes can still work because **classification** only needs the** correct class** to get the **largest score**.

The model **does not need perfect probability estimates** to **make useful predictions**.

Even when **probabilities are poorly calibrated**, the **ranking of classes can still be good**.

This is why Naive Bayes is often a strong baseline.

## 37. Naive Bayes and Finance

Naive Bayes is especially useful conceptually in finance because finance often involves updating beliefs under uncertainty.

Examples:

```txt
credit default prediction:
    P(default | borrower features)

fraud detection:
    P(fraud | transaction features)

regime detection:
    P(regime | market indicators)

event risk:
    P(adverse event | signals)

signal combination:
    P(up move | multiple indicators)
```

Naive Bayes gives a simple framework:

```txt
prior belief
+
evidence from features
=
posterior belief
```

This is very close to how many probabilistic decision systems are reasoned about.

---

## 38. Financial Example: Default Risk

Suppose we want:

```txt
P(default | features)
```

Features may include:

```txt
debt_to_income
payment_history_score
income_volatility
credit_utilization
```

Naive Bayes estimates:

```txt
P(default)
P(no_default)
```

$$
P(y = c)
$$


and:

```txt
P(debt_to_income | default)
P(payment_history_score | default)
...
```

$$
P(x_i | y)
$$

Then for a new borrower, it computes:

```txt
score(default)
score(no_default)
```

$$
\log{P(y = \text{default} | \text{features})} = \\
\log{P(y = \text{default})} \log{P(\text{features} | \text{default/no\_default})}
$$

and normalizes them into probabilities.

Even if features are correlated, this model gives a **transparent baseline**.

## 39. Financial Example: Market Regime Classification

Suppose classes are:

```txt
risk_on
risk_off
neutral
```

Features might be:

```txt
equity_return
credit_spread_change
volatility_change
yield_curve_slope
```

Gaussian Naive Bayes would estimate a Gaussian distribution for each feature inside each regime.

For a new market day, it computes:

```txt
P(risk_on | indicators)
P(risk_off | indicators)
P(neutral | indicators)
```

$$
P(y = c | x)
$$

This can be used as a simple probabilistic regime classifier.

The key limitation is that real market features are correlated and heavy-tailed, so Gaussian assumptions are imperfect.

Still, the model is useful as a baseline and as a reasoning tool.

## 40. Generative vs Discriminative Models

**Naive Bayes** is **generative**.

It models:

```txt
P(x | y) P(y)
```

**Logistic Regression** is **discriminative**.

It models:

```txt
P(y | x)
```

Comparison:

```txt
Naive Bayes:
    models feature distribution per class
    can work well with little data
    makes strong independence assumptions
    often fast and simple

Logistic Regression:
    models class probability boundary directly
    usually better calibrated with enough data
    does not assume conditional feature independence
```

In practice:

```txt
Naive Bayes can be strong with small data or high-dimensional sparse data.
Logistic Regression often wins when there is enough data and correlated features.
```

## 42. Naive Bayes Weaknesses

Main weaknesses:

```txt
conditional independence assumption is often false
Gaussian assumption may be wrong for continuous features
probabilities can be poorly calibrated
correlated features can be double-counted
outliers can distort means and variances
rare classes may need careful prior handling
```

In finance, these weaknesses matter a lot because:

```txt
features are often correlated
distributions are often heavy-tailed
regimes shift over time
historical frequencies may not be stable
```

So Naive Bayes should be treated as:

```txt
transparent baseline
probabilistic reasoning tool
not a final high-stakes financial model by itself
```
